# Server Manager

A modern .NET 8 WPF application for managing ARK: Survival Ascended dedicated servers. The app is built with MVVM, dependency injection, JSON-based configuration, rolling logs, SteamCMD integration, server process management, backups, scheduler, RCON support, and a plugin architecture.

## Project Structure

- `src/ServerManager/`
  - `Models/` — Domain model types for servers, backups, scheduler tasks, clusters, mods, and notifications.
  - `ViewModels/` — MVVM view models for dashboard, servers, mods, scheduler, console, backups, clusters, and settings.
  - `Views/` — WPF user controls for each main page.
  - `Services/` — Core services: config, logging, SteamCMD, process management, RCON, scheduler, backups, notifications, plugins.
  - `Helpers/` — Utility helpers for process execution and file handling.
  - `Resources/Theme.xaml` — Dark theme resources and shared styles.

## Features Implemented

- Multi-server instance management
- Start / stop / restart servers
- SteamCMD install/update/validate flow
- JSON configuration persistence
- Rolling log file writer with Serilog
- Basic plugin loading support
- Backup creation and restore support
- Scheduler task model with cron expressions
- RCON client shell integration
- Tray icon support and minimize-to-tray behavior
- Dark theme UI with sidebar navigation

3. Build the `ServerManager` project.
4. Run the application.

## Sample Config File

A sample config file is included in `src/ServerManager/asa-config.sample.json`.

## Logging Implementation

Logging uses Serilog with a rolling daily file sink. Logs are written to `logs/Server-Manager-*.log` in the application directory.

## Example Plugin System

A sample plugin contract is defined in `Services/IPlugin.cs` and one sample plugin implementation is included in `Plugins/SamplePlugin.cs`.

The plugin loader scans DLLs in the configured plugin directory and initializes any types that implement `IPlugin`.

## Deployment Instructions

- Build the project in Release configuration.
- Deploy the `ServerManager.exe` and dependent DLLs from `bin\Release\net8.0-windows\`.
- Ensure `asa-config.json` is present in the same folder or generated by the application.
- The application will create `logs/`, `backups/`, `plugins/`, and any SteamCMD folder automatically.

## Notes

- The server manager currently defaults to a generic ASA dedicated server executable path and parameters.
- For a live environment, configure the actual `InstallDirectory`, `ExecutableName`, and Steam app ID in the server instance settings.
